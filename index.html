Project description

If you are looking for a SPA admin template for your upcoming project then probably you come to the right place. Vadmin is a single page admin template developed by Vue.js with Laravel 8 API.It's covered most common features that you need to start a project.

 
 
 

Project Requirement

  • Any Code Editor( i prefer VS code).

  • PHP 7.4 or later.

  • Mysql 5.7 or later.

  • XAMPP/WAMP.

  • Laravel 8x.

  • Latest Node.js

Set up LaraApi Project

  • Open Mysql Server and create a new database and named it adminlv-db.

  • Open AdminApi project on editor and open .env file. you will see DB_DATABASE name set as adminlv-db.you can change the database name from here and if you want to change the DB_USERNAME and DB_PASSWORD, change those from here.

APP_ENV=local

APP_KEY=base64:oB2E1NBG07Pzp9uw1zubd7h/2A2/DC9h98ln8XyWr5g=

APP_DEBUG=false

APP_URL=http://localhost


LOG_CHANNEL=stack

LOG_LEVEL=debug


DB_CONNECTION=mysql

DB_HOST=127.0.0.1

DB_PORT=3306

DB_DATABASE=adminlv-db

DB_USERNAME=root

DB_PASSWORD=

  • Now Open terminal,go to the LaraApi project location and execute below commands     

               ->php artisan migrate

               ->php artisan db:seed

               ->php artisan serve

  • After successful migration and database seeding you should check your mysql server. you will find some tables with seed data in adminlv-db database and from here your api is ready to go.

  • Now go to LaraApi->storage->logs location, you will find here daily log files which will generate if any error occurs while running your app.

  • This project used role based authorization. Through migration data you will get two roles, Admin and User. You can add more role from Role List menu after running the Client project. These roles are used in middlewares for authorization.Currently api has two middleware for authorization, one is SuperAdminAuth.php for super admin users and CommonAuth.php for all users,you will find those on LaraApi->app->Http->Middleware location.

  • This API project is completely seperate, so you can check it by Postman or Insomnia or any other Api client.To check api you should go first on http://localhost:8000/api/Users/GetLoginInfo(POST method) and execute this by username and password,

Set up VueClient Project

  • Open AdminClient on Editor.On terminal first execute "npm install", it will add node modules to project then execute "npm run serve". Now the project will be running at http://localhost:8080/.
  • Now open VueClient->src->config.js file, you should see a hostname here which is set to http://localhost:8000. Actually our LaraApi project is running here.Change this url in case of actual hosting.
